# 12-week development roadmap

| Week | Outcome | Exit evidence |
|---|---|---|
| 1 | Product boundaries, threat model, ADRs, local stack | approved architecture and abuse cases |
| 2 | Tenant schema, migrations, JWT sessions, RBAC | tenant-isolation integration tests |
| 3 | Lead APIs and sales scoring | explainable score fixtures and CRUD UI |
| 4 | Research agent and source controls | cited research evaluation set |
| 5 | Email agent, templates, suppression policy | draft quality rubric and approval UI |
| 6 | WhatsApp adapter and conversation events | Twilio sandbox flow, opt-out tests |
| 7 | LangGraph durable checkpoints and workers | crash/resume and idempotency tests |
| 8 | Reporting service and object storage | verified PDF/dashboard artifacts |
| 9 | n8n CRM/schedule workflows | signed callbacks and replay-safe flows |
| 10 | Metrics, traces, budgets, alerting | SLO dashboard and failure runbooks |
| 11 | Security hardening and load/resilience tests | scan reports, restore drill, capacity model |
| 12 | AWS canary release and portfolio polish | release checklist, demo, architecture narrative |

After week 12: add enterprise SSO/SCIM, regional data residency, per-tenant keys, advanced evaluation, dedicated tenant deployments, and event-warehouse analytics based on validated demand.
